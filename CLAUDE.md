# 项目说明（CLAUDE.md）

本仓库用于存放一个用于秋招场景的浏览器用户脚本，核心是 `resume_autofill.user.js`。脚本从本地 `profile.json` 读取个人简历信息，在各大招聘网站的在线表单中自动填写常见字段。

---

## 项目结构

- `resume_autofill.user.js`
  - Tampermonkey 用户脚本主体
  - 核心模块：
    - 数据读写：通过 `GM_getValue` / `GM_setValue` / `GM_deleteValue` 管理本地存储（键为 `resume_profile_v1`）
    - 表单赋值工具：`setNativeValue`、`triggerAll`、`fillEl`
    - 定位与填充策略：
      - `fillByLabelText`：通过 label 文本、兄弟节点、父容器等方式寻找对应输入框
      - `fillByPlaceholder`：通过 placeholder 关键字兜底匹配
      - `fill`：组合上述策略，优先 label 匹配，其次 placeholder
    - 各业务模块填写函数：
      - `fillBasic`：个人信息
      - `fillEducation`：教育经历
      - `fillInternship`：实习经历
      - `fillLanguages`：语言能力
      - `fillCertificates`：证书
      - `fillSkills`：技能
      - `fillFamily`：家庭情况（注意姓名 label 复用问题）
    - 主入口：`autoFill`，按模块依次调用上述函数
    - UI 相关：
      - 插入页面右下角浮动按钮
      - 弹出菜单（自动填写 / 导入 / 预览 / 清除）
      - 蒙层、导入面板、预览面板
      - Toast 提示组件
- `profile.json`
  - 示例配置文件，展示脚本期望的数据结构和字段命名
  - 供用户本地准备个人信息时参考

仓库中暂无其他业务代码或配置文件。

## 开发约定

在对本项目进行修改或扩展时，建议遵守以下约定：

1. **语言与依赖**
   - 仅使用原生 JavaScript，不引入第三方框架或打包工具。
   - 保持脚本为单文件结构，方便在 Tampermonkey 中直接粘贴更新。

2. **隐私与安全**
   - 不新增任何将 `profile.json` 或本地存储中的个人信息上传至远程服务器的逻辑。
   - 如确有必要引入网络请求，需在 README 中显著说明用途与目的，并经用户明确同意后再使用。

3. **兼容性与稳定性**
   - 优先在现有工具函数 (`fillByLabelText` / `fillByPlaceholder` / `fill`) 基础上扩展，而不是为单个站点重写一套完全不同的逻辑。
   - 避免对 DOM 结构做过于依赖具体层级的假设，优先使用语义化线索（label 文本、placeholder 等）。
   - 尽量不要破坏现有字段映射，以免影响已适配的网站。

4. **日志与调试**
   - 使用 `log` 封装函数输出调试信息，保持统一前缀（如 `[简历助手]`），便于在控制台过滤。
   - 避免在生产代码中加入大量多余的 console 输出，只保留对调试有价值的关键信息。

## 对 Claude 的协作说明

当你（Claude）在本仓库中帮忙开发或调试时，请特别注意以下几点：

1. **理解现有数据模型**
   - 在修改任何填写逻辑之前，先通读 `profile.json` 的结构，理解各个字段与表单 label / placeholder 的对应关系。
   - 修改字段名或结构前，应假定用户可能有自己的本地 `profile.json`，仅在用户明确要求时才建议调整结构。

2. **适配新网站时的策略**
   - 优先尝试通过：
     - label 中文文本（精确匹配或包含匹配）
     - label 的兄弟节点
     - 附近父容器中的 input / textarea
     - placeholder 关键字
     来定位输入框。
   - 若必须加入针对某个站点的特殊逻辑，应将该逻辑封装在清晰的辅助函数中，并在代码中用简短注释说明适配目标与原因。

3. **避免过度重构**
   - 由于本脚本需要被用户直接复制到 Tampermonkey 中执行，应保持结构扁平、易读、依赖少。
   - 在没有明确需求的情况下，不要大型重构或引入复杂抽象，只对实际需要的部分做局部改动。

4. **对用户体验的考虑**
   - 保持浮动按钮和菜单文案简洁清晰，避免加入与简历填写无关的功能。
   - 任何可能影响填写结果的改动，应在提交前提醒用户自行验证：
     - 是否正确填入了所有必填项
     - 是否覆盖了不该覆盖的字段

## 本地调试建议

- 在浏览器中打开目标招聘网站页面，保持开发者工具（DevTools）打开：
  - 使用元素选择器查看 label 文本、placeholder 内容和实际 DOM 结构，验证脚本中的匹配策略是否合理。
- 在 Tampermonkey 编辑器中修改并保存脚本后，刷新目标页面即可生效，无需额外构建步骤。
- 调整 `profile.json` 中的字段值时，可以先在本地编辑器中修改，再整体复制到脚本内提供的“导入个人信息”面板进行保存。

---

## 变更建议

如果需要对脚本进行较大改动（例如重构填写策略、支持全新类别的网站等），推荐流程如下：

1. 与用户确认改动范围和目标网站列表。
2. 在 README 中补充或更新“已知限制”“支持站点”等说明。
3. 在代码中：
   - 保留原有填写逻辑，逐步扩展，避免一次性大改导致回归问题。
   - 对每个新站点或新策略用简短注释标注来源和目的，便于后续维护者排查问题。
